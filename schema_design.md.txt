# Schema Design — FinTech Pipeline

## Tables

### raw_transactions
- Rows expected: ~6.3 million
- Primary key: transaction_id (auto)
- Source: PaySim CSV columns mapped directly
- Notes: [anything you noticed about the data]

### ingestion_logs
- Purpose: audit trail for every data load run
- Status values: SUCCESS / FAILED / PARTIAL

### transaction_categories
- Purpose: reference table for transaction types
- Pre-populated with 5 PaySim types + risk levels

## Column Mapping (CSV → DB)
step            → step
type            → type
amount          → amount
nameOrig        → name_orig
oldbalanceOrg   → old_balance_orig
newbalanceOrig  → new_balance_orig
nameDest        → name_dest
oldbalanceDest  → old_balance_dest
newbalanceDest  → new_balance_dest
isFraud         → is_fraud
isFlaggedFraud  → is_flagged_fraud

## Notes
- transaction_id is a surrogate key generated by SQLite on insert.
  It does not exist in the source CSV. This is intentional.
